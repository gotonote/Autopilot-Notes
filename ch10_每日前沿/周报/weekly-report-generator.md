# 自动驾驶周报生成工作流

本文档详细描述如何系统性地收集、整理和生成每周的自动驾驶技术周报。

---

## 🎯 总体流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  信息收集   │ →  │  分类整理   │ →  │  内容撰写   │ →  │  审核发布   │
│ (周一至周六)│    │   (周日)    │    │   (周日)    │    │ (周日-周一) │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 🗂️ 第二步：分类整理（周日进行）

### 2.1 信息筛选标准

对收集到的信息进行评分，筛选出TOP5热点：

| 评分维度 | 权重 | 评分标准 |
|----------|------|----------|
| 行业影响力 | 30% | 对整个行业的影响程度 |
| 技术突破性 | 25% | 技术创新的程度 |
| 时效性 | 20% | 是否本周发生/发布 |
| 关注度 | 15% | 媒体报道和讨论热度 |
| 可解读性 | 10% | 是否有足够的背景信息 |

### 2.2 信息分类

将信息按以下维度分类整理：

```markdown
## 待分类信息

### 热点候选
1. [评分] 新闻标题 - 预估重要性⭐⭐⭐⭐⭐
2. ...

### 感知技术
- ...

### 决策规划
- ...

### 政策法规
- ...

### 产业动态
- ...

### 公司动态（按公司分类）
- Tesla: ...
- Waymo: ...
- 小鹏: ...
- 华为: ...
```

## ✍️ 第三步：内容撰写

### 3.1 热点TOP5撰写要点

每个热点应包含：

1. **标题**: 简洁有力，点明核心事件
2. **摘要**: 100-150字概括事件
3. **关键信息**: 3-5个要点，使用 bullet points
4. **影响分析**: 100-200字分析对行业/技术的影响

**示例**:
```markdown
### 1️⃣ Tesla发布FSD V13重大更新，城市道路能力大幅提升
**重要性**: ⭐⭐⭐⭐⭐  
**摘要**: Tesla本周推送FSD V13版本更新，重点优化了城市复杂路况的
处理能力。据官方数据，新版本在城区场景的无接管里程提升了40%。

**关键信息**:
- V13采用全新端到端架构，参数量提升3倍
- 新增对施工区域、临时交通标识的识别能力
- 支持更自然的车道选择和超车策略
- 已向北美10万辆车推送Beta版本

**影响分析**: 此次更新标志着Tesla在端到端自动驾驶领域迈出重要一步。
新架构的部署可能加速其他厂商跟进端到端方案，行业竞争将进一步白热化。
```

### 3.2 技术趋势分析撰写

| 维度 | 分析重点 | 建议篇幅 |
|------|----------|----------|
| 感知 | 传感器技术、视觉算法、融合方案 | 200-300字 |
| 决策规划 | 预测模型、规划算法、端到端 | 200-300字 |
| 政策法规 | 各国政策变化、标准制定 | 150-200字 |
| 产业动态 | 投融资、商业化、合作并购 | 200-300字 |

### 3.3 公司动态撰写

每家公司动态应包含：
- 本周主要动态（产品发布、技术更新、运营数据等）
- 技术亮点（如有具体技术突破）
- 市场影响（对行业/竞争对手的影响）

### 3.4 下周预告撰写

基于当前趋势预测下周热点：
- **已知事件**: 已公布日程的发布会、财报、会议等
- **技术趋势**: 根据当前技术演进方向预测
- **政策预警**: 可能出台的政策变化

## 🔍 第四步：审核检查

### 4.1 内容审核清单

- [ ] 所有TOP5热点都有完整的标题、摘要、关键信息和影响分析
- [ ] 技术趋势分析涵盖感知、决策、政策、产业四个维度
- [ ] 公司动态包含Tesla、Waymo、小鹏、华为等至少4家头部企业
- [ ] 下周预告至少包含3个值得关注的事件或趋势
- [ ] 文中没有使用模板占位符（如"示例"、"XX"等）
- [ ] 所有外部链接可正常访问
- [ ] 标点符号使用规范，无错别字

### 4.2 格式检查清单

- [ ] 文件命名符合规范：`{YYYY}-W{WW}-周报.md`
- [ ] 标题格式正确：`自动驾驶技术周报 - {YYYY}年第{X}周`
- [ ] 日期范围正确
- [ ] 使用了统一的emoji图标
- [ ] 表格格式正确，无错位

## 📤 第五步：发布

### 5.1 本地提交

```bash
cd /home/admin/project/Autopilot-Notes

# 检查修改
git status

# 添加周报文件
git add ch10_每日前沿/周报/

# 提交（使用规范的commit message）
git commit -m "docs: 添加2026年第X周自动驾驶技术周报

- 本周热点TOP5: [简要列举]
- 重点关注: [简要描述]"

# 推送到远程
git push origin main
```

### 5.2 Commit Message 规范

```
type: subject

body

# type类型:
#   docs: 文档更新（周报属于此类）
#   feat: 新功能
#   fix: 修复问题
#   refactor: 重构

# 示例:
docs: 添加2026年第8周自动驾驶技术周报

- 本周热点TOP5: Tesla FSD更新、Waymo扩张、小鹏新车发布
- 重点关注: 端到端架构成为行业新焦点
- 政策: 北京发布新的自动驾驶测试规定
```

## 💡 最佳实践

1. **保持每日收集习惯**: 每天15分钟比周日突击3小时更有效
2. **建立信息源分级**: 区分核心、重要、补充信息源
3. **保留原始链接**: 每个信息都标注来源，方便回溯
4. **关注长期趋势**: 不止看本周新闻，关注连续几周的趋势变化
5. **跨周对比**: 与上周内容对比，看哪些趋势在延续
